## 三、重难点实现

### 1. PostGIS 空间邻近查询
- **问题描述**：在实现“点击查询周边数据”功能时，直接使用 Geometry 类型计算距离会导致单位不一致（度 vs 米），且早期实现使用了错误的 `ST_Cast` 语法导致 500 错误。
- **解决方案**：强制将几何对象转换为 `Geography` 类型进行计算，确保 `ST_DWithin` 以米为单位工作，并修正 SQL 函数调用方式。
- **关键代码**：
  ```python
  # 使用 func.geography 显式转换类型，确保以米为单位计算距离
  query = db.query(GeoAsset).filter(
      GeoAsset.is_sidecar == False,
      func.ST_DWithin(
          func.geography(func.ST_SetSRID(func.ST_MakePoint(GeoAsset.center_x, GeoAsset.center_y), 4326)),
          func.geography(pt_view),
          buffer # 100米
      )
  )
  ```
- **实现效果**：成功实现了点击地图任意位置，毫秒级返回周边 100 米范围内的地质点位信息。<img src="/Users/mengzh/Library/Application Support/typora-user-images/截屏2026-02-10 13.43.06.png" alt="截屏2026-02-10 13.43.06" style="zoom:50%;" />

### 2. 矢量数据自动解析管道
- **问题描述**：用户上传的 Shapefile 通常包含多个关联文件（.shp, .shx, .dbf），且缺乏统一的空间参考，手动录入效率极低。
- **解决方案**：构建文件分组逻辑，自动识别同名文件；利用 `fiona` 库读取二进制头文件信息，提取 BBox 和 CRS。
- **关键代码**：
  ```python
  # 自动读取 Shapefile 元数据
  import fiona
  with fiona.open(str(target_shp)) as src:
      bbox = src.bounds
      # 尝试解析坐标系 SRID
      if src.crs and 'init' in src.crs:
          srid = int(src.crs['init'].split(':')[1])
  
  # 存入数据库前自动更新 PostGIS Geometry 字段
  db.execute(text("""
      UPDATE geo_assets 
      SET extent = ST_SetSRID(ST_MakeEnvelope(:minx, :miny, :maxx, :maxy, :srid), :srid)
      WHERE id = :id
  """), params)
  ```
- **实现效果**：支持拖拽上传 ZIP 或 SHP 文件组，系统自动计算并存储中心点坐标，上传完成后地图自动飞行（FlyTo）至数据位置。

### 3. 天地图 API 请求签名与编码
- **问题描述**：天地图 POI 搜索接口对 URL 结构非常敏感，直接拼接 JSON 字符串会导致 404 错误。
- **解决方案**：严格按照服务端标准接口格式构造 `postStr`，并使用 `encodeURIComponent` 进行编码。
- **关键代码**：
  ```typescript
  const postObj = {
      keyWord: keyword,
      level: "11",
      mapBound: "-180,-90,180,90",
      queryType: "1",
      start: "0",
      count: "5"
  };
  // 关键：对 JSON 字符串进行 URL 编码
  const url = `https://api.tianditu.gov.cn/search?postStr=${encodeURIComponent(JSON.stringify(postObj))}&type=query&tk=${token}`;
  ```
